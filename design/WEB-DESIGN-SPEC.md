## 0. 競合分析サマリー（設計根拠）

### 主要競合の構造
| サービス | 特徴 | 弱点（＝差別化ポイント） |
|---|---|---|
| smart-hojokin.jp | 「2分で診断」。いきなりフォーム。会社名・電話番号を入力させる | LP説明なし。信頼構築ゼロでいきなり個人情報要求→離脱率高い |
| moraeru-j.biz | チェックリスト型。助成金・補助金両対応 | UIが古い。質問数が多すぎる（6問×複数チェック）。途中離脱しやすい |
| 大塚商会/東京海上 | 大手ブランド力で信頼。5分入力→無料個別相談 | 中小企業オーナーには「大手感」が逆に敷居が高い |
| 弥生（資金調達ナビ） | 既存ユーザー向け。会計ソフトとの連携 | 新規ユーザーの流入導線が弱い |

### メッシの差別化設計
1. **LP→信頼構築→診断** の順番（競合は「いきなり診断」が多い）
2. **質問数5問以内**（競合は6問以上が標準。短い方がCVR高い）
3. **個人情報なしで結果表示**（競合はメール・電話番号必須が多い→ゲーティングはLINE登録に集約）
4. **行政書士協業の明示**（競合の多くは「誰が対応するか」が不明）

### 統計データ（設計に反映）
- LP平均CVR: 全業種6.6%（Unbounce）
- **短いLPの方がCVR 13.5%高い**（FasterCapital）→ セクション数を絞る
- パーソナライズCTAでCVR 42%向上（HubSpot）→ 「あなたの会社」を主語に
- **動画埋め込みでCVR 86%向上**（Forbes）→ v2で検討
- 見出しの質で**CVR 3倍差**（Leadpages）→ ヘッドライン最重視
- **テスティモニアル（お客様の声）がトップLP の36%に掲載**→ 初期はなくてもOK、実績蓄積後に追加
- 士業LPで最重要: **ファーストビューに人物写真** → 安心感・信頼性の大幅UP

## 0.5. ビジュアル参考サイト（デザインクオリティの基準線）

以下のサイトを **デザインクオリティの最低ライン** として参考にする。
色やブランドをコピーするのではなく、**クオリティの水準**（余白・タイポグラフィ・人物写真の使い方・UIモックの見せ方・CTA設計・アニメーションの品質）を参考にする。

### Tier 1: 日本BtoB（直接競合ではないが同じターゲット層）
| サイト | 学ぶポイント |
|---|---|
| **SmartHR** (smarthr.jp) | Heroに人物写真+プロダクトUI。ティール1色+白。余白が広い。CTA 2種類（資料DL/無料トライアル）のハードル違い設計 |
| **ジョブカン** (jobcan.ne.jp) | 「業界No.1」+「30万社」の実績バッジがHero内に。人物写真が大きい。CTA色がオレンジで目立つ |
| **freee** (freee.co.jp) | Heroスライダーで複数訴求。「法人」「個人」の導線分岐。イラスト+写真の併用 |

### Tier 2: 海外SaaS（動的演出・インタラクションの参考）
| サイト | 学ぶポイント |
|---|---|
| **Linear** (linear.app) | ダーク背景+微細なグラデーション。スクロール連動のフェード。プロダクトUIがヒーロー |
| **Vercel** (vercel.com) | タイポグラフィの美しさ。余白の使い方。ミニマルだけどリッチ |
| **Stripe** (stripe.com) | グラデーションの品質。アニメーションが上品。信頼バッジ（ロゴカルーセル） |

### ❌ 今のLPが「しょぼい」理由（自己診断）
1. **Heroに人物要素ゼロ** → 顔が見えないサイトは信頼されない
2. **プロダクトUIを見せてない** → 「何ができるか」が分からない
3. **青グラデーションが安っぽい** → Stripeのグラデーションは美しい。うちのは「とりあえず青にした」感
4. **余白が足りない** → 情報を詰め込みすぎ
5. **アニメーションが未実装 or 雑** → SmartHRのようなスムーズなフェードインがない
6. **タイポグラフィに統一感がない** → フォントサイズ・ウェイトの階層設計が甘い

## 1. ターゲット定義

### プライマリ
- **中小企業オーナー・経営者**（従業員1〜50名）
- 年齢: 40〜65歳中心
- ITリテラシー: 中〜低。スマホ主体
- 検索行動: 「補助金 うちの会社」「補助金 もらえるか」「補助金 診断」

### ペルソナ
> 田中さん（52歳）。製造業の社長。従業員12名。
> 「補助金って聞くけど、うちに当てはまるのかわからない。調べたけど種類が多すぎて諦めた。」
> 「怪しいサイトには個人情報入れたくない。でも無料なら試してみたい。」

### ペルソナから導く設計原則
1. **不安を先に解消する**（信頼→理解→行動の順）
2. **専門用語を使わない**
3. **個人情報のハードルを最小化する**（結果表示は登録不要）
4. **「自分に関係ある」と3秒で分からせる**

### セクション構成と設計意図

```

│ Header（ロゴ + CTAボタン1つのみ）    │ ← ナビリンク不要。LP=1ページ1目的
├─────────────────────────────────────┤
│ ① Hero                              │ ← 3秒で「自分に関係ある」と分からせる
│   - ヘッドライン（最重要）           │
│   - サブヘッド（1行で価値を説明）     │
│   - CTAボタン                        │

│ ② 信頼バー                          │ ← 数字3つ。スクロール前に信頼を構築

├─────────────────────────────────────┤
│ ③ 共感セクション「こんなお悩みありませんか？」│ ← 自分ごと化
│   - 3つの具体的な悩み                │
│   - 解決策への導線（テキスト1行）     │
├─────────────────────────────────────┤
│ ④ 診断の流れ（3ステップ）           │ ← 「簡単そう」と思わせる
│   1. 5つの質問に回答                 │
│   2. 結果をその場で表示              │
│   3. 興味があれば専門家に相談        │
├─────────────────────────────────────┤
│ ⑤ 安心ポイント（3つ）              │ ← 不安要素を潰す
│   - 行政書士が監修                   │
│   - 個人情報不要で診断可能           │
│   - 完全無料                         │
├─────────────────────────────────────┤
│ ⑥ 対応補助金の例（カード3-4枚）     │ ← 「こんなのあるんだ」の発見
│   - 補助金名 / 金額 / 補助率         │
│   - 「他にも○件以上」のテキスト      │
├─────────────────────────────────────┤
│ ⑦ よくある質問（FAQ 5問）           │ ← 残りの不安を解消
├─────────────────────────────────────┤
│ ⑧ CTA（最終）                       │ ← フッター直前。決断のタイミング
│   - ヘッドラインの言い換え           │
│   - CTAボタン                        │
├─────────────────────────────────────┤
│ Footer（運営情報 / プライバシーポリシー / 特商法）│
└─────────────────────────────────────┘
```

### セクション数: 8（＋Header/Footer）
- 競合の多くは10セクション以上 → **短い方がCVR高い統計に基づき絞る**
- 各セクション=1メッセージ。複合しない

### 削除したもの（v1からの変更）
- ❌ 「対応補助金一覧」の詳細表示 → 例を3-4枚に絞る。全一覧は診断後に表示
- ❌ FAQ 7問 → 5問に削減（短い方がいい）

### 追加したもの（リサーチから）
- ✅ ヘッダーにCTAボタン（スクロール不要でアクション可能）
- ✅ 安心ポイントを独立セクション化（「個人情報不要」が競合との最大差別化）
- ✅ モバイル固定CTAバー（後述）

## 5. ヘッドライン（最重要）

**統計: ヘッドラインの質でCVR 3倍差（Leadpages）。ここに一番時間をかける。**

### 構造
```
[メイン] あなたの会社が使える補助金、1分で見つかります
[サブ]  5つの質問に答えるだけ。個人情報の入力は不要です。
```

### ヘッドラインのルール
1. **「あなた（の会社）」を主語にする**（パーソナライズCTAでCVR 42%UP）
2. **具体的な数字を入れる**（「1分」「5つの質問」）
3. **不安を先に潰す**（「個人情報不要」をサブヘッドに）
4. **1行で収める**（モバイルで2行までが限界）

### NG ヘッドライン
```
❌ "補助金診断ツール"（サービス名は見出しじゃない）
❌ "AI が最適な補助金をマッチング"（横文字・技術アピール）
❌ "今すぐ無料診断！"（煽り）
❌ "補助金・助成金の活用をお考えの企業様へ"（企業様=お堅い）
```

## 6. コピーライティング

### トーン
- **ですます調。**
- 主語は「あなたの会社」「御社」ではなく「あなた」
- 専門用語は使わない。使う場合はカッコで即説明
- 金額は「万円」表記（¥5,000,000 → 500万円）

### ○ / ✗ 対照表
| ✗ | ○ | 理由 |
|---|---|---|
| AIが最適な補助金をレコメンド | あなたの会社に合った補助金を自動で見つけます | 横文字排除 |
| わずか60秒で完了 | 5つの質問に答えるだけ。約1分で結果が出ます | 具体性 |
| 今すぐ無料診断！ | まずは無料で診断してみませんか？ | 煽らない |
| ソリューションを提供 | お手伝いをします | 人間の言葉で |
| 採択確率○○% | 難易度: ★☆☆（比較的申請しやすい） | 確率は約束できない |
| 御社のDX推進を支援 | 業務効率化のための補助金もあります | DX禁止 |

### 使用推奨ワード
無料 / 簡単 / 安心 / 専門家 / 実績 / あなたの会社 / まずは / 相談

### 使用禁止ワード
AI / アルゴリズム / レコメンド / ソリューション / DX / マッチング / 最適化
「今だけ」「限定」「急いで」「お急ぎください」（煽り系全般）

## 7. CTAボタン

### デザイン仕様
```css
.cta-button {
  background-color: #E8772E;
  color: #FFFFFF;
  font-family: 'Noto Sans JP', sans-serif;
  font-weight: 700;
  font-size: 17px;               /* mobile */
  padding: 16px 48px;
  border-radius: 12px;           /* フル角丸にしない。信頼系=8-12px */
  border: none;
  cursor: pointer;
  transition: all 0.2s ease;
  box-shadow: 0 2px 8px rgba(232,119,46,0.2);
}
.cta-button:hover {
  background-color: #D06A28;
  box-shadow: 0 4px 16px rgba(232,119,46,0.3);
  transform: translateY(-1px);   /* 微動のみ。派手にしない */
}
```

### ラベル
```
✅ "無料で診断する"（メインCTA）
✅ "補助金を調べてみる"（サブCTA）
❌ "今すぐスタート！"
❌ "無料診断はこちら →"
❌ "Start"
```

### 配置ルール
| 位置 | 種類 | 備考 |
|---|---|---|
| Hero直下 | プライマリ | 最も目立つ |
| 安心ポイント直下 | プライマリ | 不安解消→即アクション |
| フッター直前 | プライマリ | 最終決断ポイント |
| モバイル画面下部 | 固定バー | スクロールしても常に見える |

- **CTA合計: 3箇所 + モバイル固定1** 
- 固定バーは背景白 + 薄いtop-shadow。CTAボタン+「無料・1分で完了」テキスト

### shimmer効果（復活・条件付き）
- Hero直下のメインCTAにのみ適用
- **1回だけ再生（ループしない）。** ページ表示後2秒後に1回shimmer → 停止
- shimmerは白の光が左→右に流れるタイプ（幅50%、opacity 0.3）
- 他のCTAには適用しない（メインCTAだけ特別感を出す）

## 8. 信頼要素（士業LP成功パターンから）

### 必須実装（リサーチ結果: 士業LPでCVR に最も影響する要素群）

| 要素 | 実装方法 | 優先度 |
|---|---|---|
| 信頼バー（数字3つ） | カウントアップ表示。セクション②に配置 | 最重要 |
| 行政書士提携の明示 | 「○名の行政書士と提携」バッジ | 最重要 |
| 「無料」の明示 | ページ内3回以上。Hero/安心ポイント/CTA | 最重要 |
| 「個人情報不要」の明示 | サブヘッド + 安心ポイント | 最重要（競合との差別化） |
| 運営会社情報 | フッター | 必須 |
| プライバシーポリシー | フッターリンク | 必須 |
| SSL/HTTPSバッジ | 安心ポイント内 or フッター | あれば |

### 将来追加（実績蓄積後）
| 要素 | タイミング |
|---|---|
| お客様の声（テスティモニアル） | 診断実績50件以上 |
| 担当行政書士のプロフィール写真 | 提携先と合意後 |
| 導入事例（ストーリー型） | 成功事例3件以上 |
| メディア掲載実績 | 掲載後 |

### ファーストビューの人物要素
- **士業LPでは人物写真がCVRに大きく影響**（ZEROラボ、StockSun共通見解）
- SmartHR/ジョブカン/freee全てHeroに人物がいる。**人物なしのHeroは「しょぼく見える」最大の原因**
- 初期は**高品質イラスト（ビジネスパーソン風）で代替** → unDraw / Storyset / Blush 等から選定
- 提携行政書士の写真が使えるようになったら即差し替え

- SmartHR: ダッシュボード画面のモック
- ジョブカン: 勤怠管理画面のモック
- freee: 確定申告画面のモック
- **うちも診断結果画面のモックアップ（スマホ型フレーム）をHeroに配置する**
- Figma or スクショベースで作成。実際のUIでなくてOK、「こういう結果が出る」のイメージ図
- モックアップは**少し傾けて影をつける**（flat配置は安っぽい。perspective mockupにする）

## 9. アニメーション

### 方針: 「控えめ」ではなく「上品」。クオリティの高いアニメーションは信頼感を上げる。
- **「アニメなし」はしょぼく見える原因。** SmartHR/freeeも全セクションにフェードインがある
- **雑なアニメーション > アニメなし > 上品なアニメーション** の順でダメ
- Framer Motionで**スムーズなeasing**（ease-out, spring）を使う。linearは機械的で安っぽい

| 要素 | アニメーション | Duration | トリガー |
|---|---|---|---|
| セクション表示 | opacity: 0→1（Fade Inのみ） | 0.6s | inView |
| 信頼バーの数字 | カウントアップ | 1.5s | inView（once） |
| 診断ステップ | 順番にFade In | 0.5s each | inView, delay 0.15s |
| 補助金カード | Fade In | 0.4s each | inView, delay 0.1s |
| FAQ開閉 | height auto + opacity | 0.3s | click |
| CTA hover | 色変化 + 微shadow + translateY(-1px) | 0.2s | hover |
| モバイル固定バー | 下からスライド | 0.3s | scroll > 500px |

### OK（品質が高ければ）
- ✅ Fade In + 軽い translateY(20px)（SmartHR/freeeで使用。自然な登場感）
- ✅ CTAボタンの shimmer効果（控えめなら。1回だけ、ループしない）
- ✅ ホバーでカードが浮く（translateY(-4px) + shadow拡大）
- ✅ 信頼バー背景の微グラデーション（Stripe風。動かない静的グラデ）
- ✅ スクロールに連動した軽いフェード（intersection observer）
- ✅ Heroのプロダクトモックに軽いフロート（上下3px、8秒周期）

### NG
- ❌ Blob装飾 / フローティング絵文字
- ❌ スクロールジャック
- ❌ ページ読み込みスプラッシュ
- ❌ テキストの1文字ずつ表示
- ❌ 背景全体が動くグラデーション
- ❌ 過度なパララックス（軽いものはOK）

## 10. レイアウト・余白

## 12. 診断フォームUI（LP内の最重要パーツ）

### 設計原則
- **質問数: 5問以内**（競合は6問以上→短い方が完了率高い）
- **個人情報は聞かない**（名前・電話番号・メール不要で結果表示）
- **プログレスバー表示**（「あと○問」を常に見せる）
- **1画面1問**（複数問を1画面に詰め込まない）

### 質問の流れ
```
Q1. 業種を選んでください（プルダウン or カード選択）
Q2. 従業員数を教えてください（選択肢4つ）
Q3. 年間売上の規模を教えてください（選択肢4つ）
Q4. 興味のある分野を選んでください（複数選択可）
    □ 設備投資  □ IT導入  □ 人材採用・育成  □ 販路拡大  □ 事業転換
Q5. 所在地（都道府県）を選んでください
```

### フォームUIスタイル
- 選択肢はボタン型（ラジオボタンの見た目ではなく、タップしやすいカード型）
- 選択済み: Primary色の枠線 + 薄いPrimary Light背景
- 未選択: Border色の枠線
- 戻るボタン: テキストリンク（「← 前の質問に戻る」）
- 次へボタン: Primary色（Accent色はCTAのみ）

### 結果画面
- 該当する補助金をカード表示
- 各カードに「補助金名 / 最大金額 / 補助率 / 難易度」
- 下部に**LINE登録CTA**（「詳しい情報をLINEでお届けします」）
- LINE登録がゲーティングポイント。結果概要は無料、詳細はLINE登録後

## 18. A/Bテスト計画（リリース後）

統計: A/Bテストの87.5%は有意な差が出ない。テストは仮説を絞って1つずつ。

| 優先度 | テスト対象 | 仮説 |
|---|---|---|
| 1 | ヘッドラインの文言 | 「1分で見つかる」vs「5つの質問だけ」どちらがCTR高いか |
| 2 | CTAボタンの色 | オレンジ vs 緑 |
| 3 | 診断結果のゲーティング位置 | 全結果無料 vs 上位3件無料+残りはLINE登録 |
| 4 | Hero の人物要素 | イラスト vs 写真（提携先の写真が使えるようになったら） |
